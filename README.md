## 🤖 AI-Driven Development

I treat AI as a core part of my engineering workflow, not an afterthought. My **agentic coding lifecycle** typically looks like:

- **Plan** → break work into scoped, dependency-ordered steps with AI before writing a line of code
- **Build** → pair with agentic coding tools to implement features and refactors at speed
- **Test** → generate and maintain unit/integration/E2E coverage alongside the code
- **Review** → run AI-assisted code review to catch bugs, edge cases, and regressions early
- **Document** → keep knowledge bases and docs in sync automatically as behavior changes

**Why it matters:** this approach compresses feedback loops, reduces boilerplate toil, and lets me focus energy on architecture and product decisions — directly boosting development velocity and product quality.

**Tools & practices:** Claude Code, Cursor, GitHub Copilot, MCP (Model Context Protocol), prompt engineering, AI-assisted code review, and agent-driven test/documentation workflows.
